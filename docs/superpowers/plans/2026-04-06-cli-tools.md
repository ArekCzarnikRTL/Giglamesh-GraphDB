# Feature 31: CLI Tools Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A Kotlin/Clikt CLI for GraphMesh that uses a typed graphql-kotlin Ktor client to talk to the backend, supports TABLE and JSON output, and ships as a Fat-Jar plus bin/lib install layout — including the backend additions needed to make Config and Document-Hierarchy commands functional.

**Architecture:** CLI lives inside the existing single-module Gradle project under `src/main/kotlin/com/agentwork/graphmesh/cli/`, with its own `main` function that never touches the Spring context. The `com.expediagroup.graphql` Gradle plugin generates typed GraphQL query classes from `.graphql` files plus a consolidated SDL (produced by a custom `assembleCliSchema` task). Commands go through a thin `GraphQlGateway` wrapper around `GraphQLKtorClient`, map responses to `*View` records, and send them to a pluggable `Output` (TABLE/JSON).

**Tech Stack:** Kotlin 2.2, Clikt 5.x, graphql-kotlin-ktor-client 8.x, Ktor CIO 3.x, Jackson (reuse existing `jackson-module-kotlin`), JUnit 5 + MockK (existing), Spring Boot 4 GraphQL controllers (backend additions only).

**Reference documents:**
- Spec: `docs/superpowers/specs/2026-04-06-cli-tools-design.md`
- Feature doc: `docs/features/31-cli-tools.md`

**Project conventions confirmed by reading the codebase:**
- Package root: `com.agentwork.graphmesh`
- GraphQL controllers live in `src/main/kotlin/com/agentwork/graphmesh/api/` (e.g., `CollectionController.kt`, `DocumentController.kt`, `ExplainabilityController.kt`)
- Input DTOs live in `src/main/kotlin/com/agentwork/graphmesh/api/InputTypes.kt` OR next to the controller that uses them (e.g., `GraphRagInput` in `GraphRagController.kt`) — this plan mirrors whichever pattern matches the surrounding file
- GraphQL schemas are split across `src/main/resources/graphql/*.graphqls` using `extend type Query` / `extend type Mutation`
- Tests use pure MockK (`io.mockk`) + `kotlin.test` — no MockMvc GraphQL wiring; controllers are invoked directly with mocked services
- Existing `generateJava` (DGS codegen) task is configured but unused (`graphql-client/` is empty); we are introducing a second codegen path — `graphqlGenerateClient` from the graphql-kotlin plugin — they do not conflict
- `CreateCollectionInput` has `tags: List<String>?` and `metadata: List<KeyValueInput>?`
- `UploadDocumentInput.content: String` holds Base64 (server-side `Base64.getDecoder().decode(...)`)
- `ConfigService` has no `findAll()` — this plan adds one that iterates `ConfigType.values()`
- `ProvenanceService` has no document-hierarchy traversal — this plan introduces traversal via `LibrarianService.findById()` + `findChildren()` (the hierarchy lives in the Document tree via `parentId`)

**Files:**

- [ ] **Step 1: Register the `assembleCliSchema` task**

Append to the bottom of `build.gradle.kts`:

```kotlin
val assembleCliSchema by tasks.registering {
    group = "cli"
    description = "Concatenates all graphql/*.graphqls files into a single SDL for CLI codegen."
    val inputDir = layout.projectDirectory.dir("src/main/resources/graphql")
    val outputFile = layout.buildDirectory.file("generated/cli-schema/schema.graphqls")
    inputs.dir(inputDir)
    outputs.file(outputFile)
    doLast {
        val out = outputFile.get().asFile
        out.parentFile.mkdirs()
        val files = inputDir.asFile.listFiles { f -> f.extension == "graphqls" }
            ?: error("No .graphqls files found in ${inputDir.asFile}")
        // schema.graphqls defines base Query/Mutation — it must come first
        val sorted = files.sortedWith(
            compareByDescending<java.io.File> { it.name == "schema.graphqls" }
                .thenBy { it.name }
        )
        out.writeText(sorted.joinToString("\n\n") { it.readText() })
    }
}
```

Create `src/main/kotlin/com/agentwork/graphmesh/api/ConfigGraphQlController.kt`:

```kotlin
package com.agentwork.graphmesh.api

import com.agentwork.graphmesh.config.ConfigItem
import com.agentwork.graphmesh.config.ConfigService
import com.agentwork.graphmesh.config.ConfigType
import org.springframework.graphql.data.method.annotation.Argument
import org.springframework.graphql.data.method.annotation.MutationMapping
import org.springframework.graphql.data.method.annotation.QueryMapping
import org.springframework.stereotype.Controller

@Controller
class ConfigGraphQlController(
    private val configService: ConfigService
) {

    @QueryMapping
    fun configKeys(@Argument type: String?): List<ConfigEntryView> {
        val typeEnum = type?.let { runCatching { ConfigType.valueOf(it) }.getOrNull() }
        return configService.findAll(typeEnum).map { ConfigEntryView.from(it) }
    }

    @QueryMapping
    fun configValue(@Argument key: String, @Argument type: String): ConfigEntryView? {
        val typeEnum = runCatching { ConfigType.valueOf(type) }.getOrNull() ?: return null
        return configService.findByTypeAndKey(typeEnum, key)?.let { ConfigEntryView.from(it) }
    }

    @MutationMapping
    fun setConfig(
        @Argument key: String,
        @Argument value: String,
        @Argument type: String
    ): ConfigEntryView {
        val typeEnum = ConfigType.valueOf(type)
        val existing = configService.findByTypeAndKey(typeEnum, key)
        val item = (existing ?: ConfigItem(
            id = "$type:$key",
            type = typeEnum,
            key = key,
            value = value
        )).copy(value = value)
        return ConfigEntryView.from(configService.save(item))
    }
}

data class ConfigEntryView(
    val id: String,
    val type: String,
    val key: String,
    val value: String,
    val version: Int
) {
    companion object {
        fun from(item: ConfigItem) = ConfigEntryView(
```

```kotlin
package com.agentwork.graphmesh.api

import com.agentwork.graphmesh.librarian.Document
import com.agentwork.graphmesh.librarian.LibrarianService
import org.springframework.graphql.data.method.annotation.Argument
import org.springframework.graphql.data.method.annotation.QueryMapping
import org.springframework.stereotype.Controller

@Controller
class DocumentHierarchyController(
    private val librarianService: LibrarianService
) {

    @QueryMapping
    fun documentHierarchy(
        @Argument collectionId: String,
        @Argument documentId: String
    ): DocumentNodeView? {
        val root = librarianService.findById(documentId) ?: return null
        if (root.collectionId != collectionId) return null
        return build(root, visited = mutableSetOf(), depth = 0)
    }

    private fun build(doc: Document, visited: MutableSet<String>, depth: Int): DocumentNodeView {
        if (depth >= MAX_DEPTH) {
            return DocumentNodeView(doc.id, doc.title, doc.type.name, emptyList())
        }
        visited += doc.id
        val children = librarianService.findChildren(doc.id)
            .filter { it.id !in visited }
            .map { build(it, visited, depth + 1) }
        return DocumentNodeView(
            id = doc.id,
            title = doc.title,
            type = doc.type.name,
            children = children
        )
    }

    companion object {
        private const val MAX_DEPTH = 10
    }
}

data class DocumentNodeView(
    val id: String,
    val title: String,
    val type: String,
    val children: List<DocumentNodeView>
)
```

```kotlin
package com.agentwork.graphmesh.cli.output

/**
 * Human-readable ASCII formatter. Renders lists as bordered tables with
 * per-column auto-width (truncated at [maxColWidth]) and single objects as
 * key/value blocks. All output goes through the [sink] callback — the CLI
 * wires this to Clikt's `echo`.
 */
class TableOutput(
    private val sink: (String) -> Unit,
    private val maxColWidth: Int = 60
) : Output {

    // -- List rendering ---------------------------------------------------

    override fun writeCollections(items: List<CollectionView>) {
        if (items.isEmpty()) { sink("(no collections)"); return }
        renderTable(
            headers = listOf("ID", "Name", "Tags", "Created"),
            rows = items.map { listOf(it.id, it.name, it.tags.joinToString(","), it.createdAt) }
        )
    }

    override fun writeDocuments(items: List<DocumentView>) {
        if (items.isEmpty()) { sink("(no documents)"); return }
        renderTable(
            headers = listOf("ID", "Title", "MIME", "Type", "State", "Created"),
            rows = items.map { listOf(it.id, it.title, it.mimeType, it.type, it.state, it.createdAt) }
        )
    }

    override fun writeConfigEntries(items: List<ConfigEntryView>) {
        if (items.isEmpty()) { sink("(no config entries)"); return }
        renderTable(
            headers = listOf("Type", "Key", "Value", "Version"),
            rows = items.map { listOf(it.type, it.key, it.value, it.version.toString()) }
        )
    }

    override fun writeExplanationSessions(items: List<QuestionExplanationView>) {
        if (items.isEmpty()) { sink("(no sessions)"); return }
        renderTable(
            headers = listOf("URI", "Question", "Mechanism", "Timestamp"),
            rows = items.map { listOf(it.uri, it.queryText, it.mechanism, it.timestamp) }
        )
    }

    // -- Stubs for key/value and tree rendering (Tasks 2.6 and 2.7) -------

    override fun writeCollectionCreated(item: CollectionView) = sink("Collection created: ${item.id}")
    override fun writeCollectionDeleted(id: String) = sink("Collection deleted: $id")
    override fun writeDocumentInfo(doc: DocumentInfoView) = sink(doc.toString())          // Task 2.6
    override fun writeDocumentUploaded(doc: DocumentInfoView, sizeBytes: Long) = sink("Uploaded ${doc.id} (${sizeBytes} bytes)")
    override fun writeGraphRag(result: GraphRagResponseView) = sink(result.toString())    // Task 2.6
    override fun writeDocRag(result: DocRagResponseView) = sink(result.toString())        // Task 2.6
    override fun writeNlp(result: NlpResponseView) = sink(result.toString())              // Task 2.6
    override fun writeConfigEntry(entry: ConfigEntryView) = sink("${entry.type}:${entry.key} = ${entry.value}")
    override fun writeExplanationChain(chain: ExplanationChainView, maxAnswerChars: Int) = sink(chain.toString()) // Task 2.6
    override fun writeDocumentHierarchy(root: DocumentNodeView) = sink(root.toString())   // Task 2.7
    override fun writeMessage(message: String) = sink(message)

    // -- Internal helpers --------------------------------------------------

    private fun renderTable(headers: List<String>, rows: List<List<String>>) {
        val all = listOf(headers) + rows
        val widths = IntArray(headers.size)
        for (row in all) {
            for ((i, cell) in row.withIndex()) {
                val truncated = truncate(cell)
                if (truncated.length > widths[i]) widths[i] = truncated.length
            }
        }
        val sep = widths.joinToString(separator = "-+-", prefix = "+-", postfix = "-+") { "-".repeat(it) }
        val headerLine = headers.mapIndexed { i, h -> truncate(h).padEnd(widths[i]) }
            .joinToString(separator = " | ", prefix = "| ", postfix = " |")
        sink(sep)
        sink(headerLine)
        sink(sep)
        for (row in rows) {
            val line = row.mapIndexed { i, cell -> truncate(cell).padEnd(widths[i]) }
                .joinToString(separator = " | ", prefix = "| ", postfix = " |")
            sink(line)
        }
        sink(sep)
    }
```

```kotlin
package com.agentwork.graphmesh.cli

import com.expediagroup.graphql.client.ktor.GraphQLKtorClient
import com.expediagroup.graphql.client.types.GraphQLClientRequest
import com.github.ajalt.clikt.core.CliktError
import io.ktor.client.HttpClient
import io.ktor.client.engine.cio.CIO
import io.ktor.client.plugins.HttpTimeout
import java.net.URL

/**
 * Thin wrapper around [GraphQLKtorClient] that injects the Bearer token header
 * and maps GraphQL errors to [CliktError]s so Clikt can render them cleanly.
 */
class GraphQlGateway private constructor(
    private val client: GraphQLKtorClient,
    private val token: String
) : AutoCloseable {

    constructor(cfg: CliConfig) : this(
        client = GraphQLKtorClient(
            url = URL(cfg.endpoint),
            httpClient = HttpClient(CIO) {
                install(HttpTimeout) { requestTimeoutMillis = 60_000 }
            }
        ),
        token = cfg.token
    )

    suspend fun <T : Any> execute(request: GraphQLClientRequest<T>): T {
        val response = client.execute(request) {
            if (token.isNotBlank()) {
                headers.append("Authorization", "Bearer $token")
            }
        }
        val errors = response.errors
        if (!errors.isNullOrEmpty()) {
            throw CliktError("GraphQL error: " + errors.joinToString(", ") { it.message ?: "unknown" })
        }
        return response.data ?: throw CliktError("GraphQL response contained no data")
    }

    override fun close() = client.close()
```

```kotlin
package com.agentwork.graphmesh.cli

import com.expediagroup.graphql.client.types.GraphQLClientRequest
import com.github.ajalt.clikt.core.CliktError
import kotlin.reflect.KClass

/**
 * Minimal in-memory gateway for command tests. Each command test registers
 * a pair: (request KClass) -> (factory producing a canned response or throwing).
 */
class FakeGateway private constructor(
    private val handlers: Map<KClass<out GraphQLClientRequest<*>>, suspend (GraphQLClientRequest<*>) -> Any>
) {
    @Suppress("UNCHECKED_CAST")
    suspend fun <T : Any> execute(request: GraphQLClientRequest<T>): T {
        val handler = handlers[request::class]
            ?: throw CliktError("FakeGateway: no handler for ${request::class.simpleName}")
        return handler(request) as T
    }

    class Builder {
        private val handlers = mutableMapOf<KClass<out GraphQLClientRequest<*>>, suspend (GraphQLClientRequest<*>) -> Any>()
        fun <R : GraphQLClientRequest<T>, T : Any> on(
            requestClass: KClass<R>,
            handler: suspend (R) -> T
        ): Builder = apply {
            @Suppress("UNCHECKED_CAST")
            handlers[requestClass] = { req -> handler(req as R) }
        }
        fun build() = FakeGateway(handlers.toMap())
    }
}
```

```kotlin
                        arguments = a.arguments?.map { it.key to it.value } ?: emptyList(),
                        observation = a.observation
                    )
                },
                synthesis = chain.synthesis?.let { SynthesisView(it.uri, it.answerText) },
                conclusion = chain.conclusion?.let { ConclusionView(it.uri, it.answerText) }
            ),
            maxAnswerChars = maxAnswer
        )
    }
}

class ExplainDocument : BaseCommand("document", "Show document hierarchy tree") {
    private val documentId by argument(help = "Document ID")
    private val collectionId by option("--collection", "-c", help = "Collection ID").required()

    override suspend fun run() {
        val result = gateway().execute(GetDocumentHierarchy(GetDocumentHierarchy.Variables(
            collectionId = collectionId, documentId = documentId
        )))
        val root = result.documentHierarchy ?: run {
            out.writeMessage("Document $documentId not found")
            return
        }
        out.writeDocumentHierarchy(toView(root))
    }

    private fun toView(node: Any): DocumentNodeView {
        // Adapt to the concrete generated nested type at each level. Because
        // GraphQL requires explicit selections per level, the generated class is
        // distinct at each depth; use reflection-free when-cast on the well-known
        // types produced by the codegen.
        return when (node) {
            is GetDocumentHierarchy.Result.DocumentNode ->
                DocumentNodeView(node.id, node.title, node.type,
                    node.children.map { toView(it) })
            // Nested levels — adjust names to match your codegen output
            else -> error("Unexpected hierarchy node type: ${node::class}")
        }
    }
}
```

> Note on the tree mapping: graphql-kotlin produces a different nested class for each level of the query selection (since GraphQL cannot express recursion). The `toView` helper above handles only the top level. For levels 2-4 you need to add `is GetDocumentHierarchy.Result.DocumentNode.Children ->` branches — **read the generated source** to get the exact class names and add one branch per level. The 4-level query from Task 3.5 covers realistic hierarchies.
